1. Cómo empezar

Esta guía es un ejemplo práctico del uso de ArgenMap v2. La misma está basada en casos de estudio en base a los cuales se concibió ArgenMap. Si buscas ayuda específica de algún método u opción, debes remitirte a la sección de desarrolladores, donde podrás hallar un listado detallado de los métodos y funciones de ArgenMap v2.

ArgenMap es un plugin para jQuery. Todos los ejemplos asumen un conocimiento básico de como se utiliza jQuery. Si no estás familiarizado, te recomendamos que visites el sitio de jQuery para que te enteres rápidamente cómo se utiliza y las ventajas que representa

En estas guías se asume que para ejecutar las pruebas se cuenta con un HTML bien redactado y que lleva incluido el script de jQuery y el de ArgenMap v2 como este: 

<!DOCTYPE html>
<html>
  <head>
  <!-- jQuery, vinculado directo a cdn de google -->
  <script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js"></script>
  <!-- ArgenMap v2, vinculado directo desde el Instituto Geográfico Nacional -->
  <script type="text/javascript" src="http://www.ign.gob.ar/argenmap2/argenmap.jquery/argenmap.jquery.min.js"></script>
  </head>
  <body>
    <div id="prueba" style="height:450px;"></div>
  </body>
</html>

Tal vez tu página no se ve así, pero lo importante es que incluyas los scripts de jQuery y de ArgenMap en el head de tu página: 

<head>
...
<!-- jQuery, vinculado directo a cdn de google -->
<script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.7.2/jquery.min.js"></script>
<!-- ArgenMap v2, vinculado directo desde el Instituto Geográfico Nacional -->
<script type="text/javascript" src="http://www.ign.gob.ar/argenmap2/argenmap.jquery/argenmap.jquery.min.js"></script>
...
</head>

...luego colocas un DIV donde quieres mostrar un mapa. Es importante que, sea por CSS o por estilo en el DIV, la altura esté definida. Recuerda asignarle un ID o alguna clase por la cual luego puedas hacer referencia con jQuery

...
<div id="mapa" style="height:400px"></div>
...

Y por último simplemente haces la llamada desde un script en la página. Teniendo en cuenta que las librerías pueden demorar en cargarse, es mejor ejecutar cualquier llamada una vez que la página esté cargada. 

<script>
//con esta llamada, el código se ejecuta
//cuando la página terminó de cargarse
$(document).ready(function(){
  //... líneas de código
  $('#mapa').argenmap();
  //... más líneas de código
});
</script>

Consejo: Siempre es conveniente ejecutar cualquier script luego que la página ha terminado de cargarse. Por este motivo se recomienda que las llamadas se hagan de esta manera para evitar errores difíciles de encontrar.

Código

Todo el código que se encuentra en estas guías se refiere a JavaScript y/o HTML.
A menudo se darán ejemplos con un botón para poder ejecutar el código y ver su resultado.
Este es un script tag tal como esta insertado en esta página (sólo el selector es distinto).

<script>
  //al ejecutar este código se verá
  //el resultado en el cuadro de ejemplo
  $('#mapa').argenmap();
</script>

Resultado

El resultado del código se mostrará siempre en un DIV con un borde punteado rojo, como el que se muestra aquí debajo.

2. Un mapa

ArgenMap es una librería para mostrar mapas de Argentina. Su función más básica será instanciar un mapa en el/los selector/es indicados con jQuery

$(selector).argenmap([opciones:Objeto])

.argenmap() inicializa un mapa en el selector indicado

Las opciones del mapa pueden ser incluidas al momento de inicializarlo mediante un objeto de JavaScript, o bien pueden ser modificadas luego a través de llamadas al mismo selector.

Utilizando un objeto con propiedades centro y zoom se puede configurar el mapa para la zona que se quiere enfocar.
Este es un ejemplo de como inicializar un mapa con distintas opciones: 
<script>
//inicialización básica
$('#prueba').argenmap();

//inicialización con centro en el Aconcagua
$('#prueba').argenmap(
  {
    centro: [-32.6533333,-70.0108333],
    zoom: 12
  });  Ejecutar ejemplo

//vista del Parque Centenario
$('#prueba').argenmap(
  {
    centro: [-34.606942257,-58.435303252],
    zoom: 14
  });  Ejecutar ejemplo
</script>

Tipos de mapa

También puede definirse un tipo de mapa. Los valores válidos son baseign y satelital. Por defecto, el tipo de mapa es baseign

<script>
//inicialización con centro en el Aconcagua
$('#prueba').argenmap({
  tipo: 'satelital',
  centro: [-32.6533333,-70.0108333],
  zoom: 12
});  Ejecutar ejemplo

//vista del Parque Centenario, predeterminado
$('#prueba').argenmap({
  tipo: 'baseign',
  centro: [-34.606942257,-58.435303252],
  zoom: 14
});  Ejecutar ejemplo

//vista del Parque Centenario, satelite
$('#prueba').argenmap({
  tipo: 'satelital',
  centro: [-34.606942257,-58.435303252],
  zoom: 14
});  Ejecutar ejemplo
</script>

$(selector).centro(lat, lon)

Luego de instanciar un mapa, puede usarse .centro(lat,lon) para centrar el mapa donde se desée.

lat (requerido): número indicando los grados decimales de latitud

lon (requerido): número indicando los grados decimales de longitud

quiere visualizar el mapa.

Lectura/escritura

Si no se provee latitud ni longitud, el método devuelve un array con las coordenadas actuales del centro del mapa.

Nota: al ejecutar la llamada en modo lectura (sin parámetros) la función rompe la encadenabilidad de jQuery, es decir, solo devuelve el resultado para la primer coincidencia del selector

<script>
//instancia de mapa y
//la llamada puede encadenarse:
$('#prueba').argenmap().centro(-34.872,-57.965);
Ejecutar ejemplo

//una vez instanciado, podemos simplemente
//hacer la llamada, vista del Parque Centenario
$('#prueba').centro(-34.6069,-58.4353);
Ejecutar ejemplo

//lago Nahuel Huapi
$('#prueba').centro(-41.1217,-71.3095);
Ejecutar ejemplo
</script>

$(selector).zoom(zoom)

Acerca o aleja la vista del mapa sin cambiar de posición.

zoom (requerido): 0 - 19 número indicando el nivel de zoom al que se quiere visualizar el mapa.

<script>
//vista del Parque Centenario
$('#prueba').argenmap()
  .centro(-34.6069,-58.4353).zoom(14);
Ejecutar ejemplo

//para alejar, se disminuye el nivel de zoom
$('#prueba').zoom(12);
Ejecutar ejemplo

//para acercar, se aumenta el nivel de zoom
$('#prueba').zoom(15);
Ejecutar ejemplo
</script>

3. Capas

Una vez instanciado el mapa, se puede optar por mostrar capas desde otros servidores o mismo del servicio WMS del Instituto Geográfico Nacional.

Todas las capas, sin importar que tipo sean, llevan una propiedad nombre para poder identificarlas luego. Es importante intentar que nombre sea único, pero no es imprescindible.
Cualquier otra operación a realizar con la capa, deberá ser referida por el nombre (removerCapa, modificarCapa, modificarTransparenciaDeCapa)

$(selector).agregarCapa(capaPredefinida)

Capas predefinidas

Las capas predefinidas son capas que pueden agregarse simplemente por su nombre, sin necesidad de ningún otro parámetro, haciendo la llamada .agregarCapa("nombre_de_capa")

Base IGN

baseign Es la capa predeterminada, actúa como capa base, pero si el mapa se inicializó como satelital puede luego agregarse esta capa.

Satelital

satelital Instancia el mapa con una base de imágenes de satélite y los datos del IGN superpuestos.

Advertencia:

No hay límite a la cantidad de veces que puede agregarse una capa, si bien puede resultar divertido, no se recomienda agregar una capa más de una vez. 
<script>
//vista del mapa normal
$('#prueba').argenmap();
Ejecutar ejemplo

//agregar capa satelital al mapa
$('#prueba').agregarCapa('satelital');
Ejecutar ejemplo

//mapa satelital, agregar capa baseign
//encadenando la llamada
$('#prueba').argenmap({tipo:satelital})
  .agregarCapa('baseign');
Ejecutar ejemplo
</script>

$(selector).agregarCapaWMS(opciones)

Las opciones que pueden pasarse son los parámetros que pueden pasarse a cualquier WMS:

url
cadena de texto indicando la url del servidor WMS. Ejemplo: http://mapa.ign.gob.ar/wms
capas
cadena de texto de las capas que se quieren pedir al servidor WMS, separadas por coma. Ejemplo: "departamentos, rios, ciudades"

Opcionales
 
nombre
cadena de texto indicando el nombre que tendrá la capa. Recuerda que es importante definir el nombre e intentar que sea único. Es el nombre que se utilizará en el panel de capas.
formato
valor por defecto: "image/png" Cadena de texto indicando un formato MIME de archivo: "image/jpeg", "image/jpg", "image/png", "image/png8", "image/gif".
transparente
valor por defecto: true Boolean indicando si la capa tiene trasparencias. Esta opción debe ser coherente con formato: solo "image/png", "image/gif" o "image/png8" pueden ser transparentes.

Existen más opciones para configurar una capa WMS, si necesitas más detalles, puedes consultar en la sección de desarrolladores:Capas WMS.
Ejemplo de capa WMS: capa de datos del IGN 
//instanciamos un mapa vacio
$('#prueba').argenmap({tipo:'blanco'});
Ejecutar ejemplo

//creamos las opciones para una capa WMS
var capaBaseIGN = {
  url: 'http://www.ign.gob.ar/wms',
  nombre: 'Capa IGN',
  capas: 'capabasesigign',
  formato: 'image/jpeg',
  esCapaBase: false,
  transparente: false
}
$('#prueba').agregarCapaWMS(capaBaseIGN);
Ejecutar ejemplo

//la misma capa, en modo transparente para superposición:
var capaTransparenteIGN = {
  url: 'http://www.ign.gob.ar/wms',
  nombre: 'Capa IGN Transparente',
  capas: 'capabasesigign',
  formato: 'image/png',
  transparente: true
}
//Atención: debes ejecutar el ejemplo anterior
//para que esta línea de un resultado
$('#prueba').agregarCapaWMS(capaTransparenteIGN);
Ejecutar ejemplo

$(selector).agregarCapaKML(opciones)

Es una capa que se importa desde un archivo .kml. Sus opciones son:

url
cadena de texto indicando donde se encuentra, relativo al archivo HTML donde se encuentra el mapa, el archivo KML.

Opcionales
 
nombre
cadena de texto indicando el nombre que tendrá la capa. Recuerda que es importante definir el nombre e intentar que sea único. Es el nombre que aparecerá en el panel de capas.
El archivo KML debe estar en el mismo protocolo, host y dominio que el archivo desde donde se instancia el mapa. De no ser así, probablemente recibas un error referente a Política del mismo origen (Same origin policy). 
var capaKml = {
  tipo: 'kml',
  url: 'prueba.kml',
  nombre: 'Capa KML'
}
$('#prueba').argenmap().agregarCapaKML(capaKml);
Ejecutar ejemplo

4. Marcadores

Trabajar con marcadores debería ser simple, esa es la idea con ArgenMap. Para crear un marcador se puede suministrar un objeto con opciones o simplemente hacer la llamada $(selector).agregarMarcador() y por defecto el marcador se agregará en el centro del mapa.

$(selector).agregarMarcador([opciones])

Para personalizar el comportamiento del marcador pueden utilizarse estas opciones

lonlat
array(lon,lat) de coordenadas donde se ubicará el marcador. Ejemplo: [-56.32,-35,84]

Opcionales
 
nombre
cadena de texto indicando el nombre que tendrá el marcador. Recuerda que si bien es opcional, es importante definir el nombre e intentar que sea único para luego poder accederlo si necesitaras modificarlo.
lon
Las coordenadas pueden proveerse por separado, nombrando cada valor por su eje (lon y lat)
lat
Las coordenadas pueden proveerse por separado, nombrando cada valor por su eje (lon y lat)
contenido
Cadena de texto con los datos que mostrará el marcador. Esta cadena acepta HTML.

$('#prueba').argenmap().agregarMarcador();
Ejecutar ejemplo

$('#prueba').argenmap().agregarMarcador({
  lon:-56,
  lat:-34
});
Ejecutar ejemplo

$('#prueba').argenmap().agregarMarcador({
  lon:-58,
  lat:-32,
  contenido: '<h2>Marcador</h2><p>Este es el contenido del marcador</p>'
});
Ejecutar ejemplo
